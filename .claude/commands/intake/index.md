# 비판적 검증 시스템 (Critical Review Protocol)

건재가 기사, 논문, 블로그 글, 리포트, 또는 any 텍스트를 붙여넣으면 아래 **7단계 비판적 검토**를 자동 실행한다.

---

## Phase 1 — 출처 검증 (Source Verification)

- 이 글의 **원저자는 누구**인가?
- **1차 자료**(연구 원문, 공식 발표, 직접 실험)인가, **2차 해석**(기자 기사, 블로그 요약, 유튜브 설명)인가?
- 원본 1차 자료가 존재한다면 **링크 또는 출처를 명시**하고, 현재 텍스트와의 해석 차이가 있는지 확인한다.
- 2차 해석일 경우, 원문 대비 **누락·왜곡·과장된 부분**이 있는지 지적한다.

### 출력 형식

```text
[출처 등급] 1차 자료 / 2차 해석 / 3차 재가공 / 출처 불명
[원저자]    이름, 소속, 전문 분야
[원본 링크] 확인 가능 여부
```

---

## Phase 2 — 이해충돌 검증 (Conflict of Interest Check)

> Phase 1에서 빠지기 쉬운 **"왜 이 사람이 이 말을 하는가?"** 를 검증한다.

- 저자 또는 발행 기관이 해당 주장으로 인해 **금전적·평판적·정치적 이익**을 얻는가?
- 스폰서, 광고주, 소속 기업과의 **이해관계**가 존재하는가?
- 특정 제품·서비스·이념을 **홍보하려는 동기**가 감지되는가?

### 출력 형식

```text
[이해충돌] 없음 / 잠재적 / 명백함
[근거]      구체적 이유 1줄 요약
```

---

## Phase 3 — 근거 검증 (Evidence Verification)

각 핵심 주장을 추출한 뒤, 개별적으로 근거 유형을 분류한다.

| 근거 등급 | 정의 | 신뢰도 |
| --------- | ---- | ------ |
| **A — 실증 데이터** | 재현 가능한 실험, 통계, 계량 데이터 | ★★★★★ |
| **B — 사례 증거** | 특정 기업·인물의 성공/실패 사례 | ★★★☆☆ |
| **C — 전문가 의견** | 해당 분야 전문가의 판단 (데이터 없이) | ★★★☆☆ |
| **D — 일반 의견** | 비전문가의 주장, 상식적 추론 | ★★☆☆☆ |
| **E — 추측/예측** | 근거 없는 미래 전망, 감정 기반 판단 | ★☆☆☆☆ |

### 추가 확인 항목

- 데이터가 있다면: **표본 크기, 연구 방법론, p-value 또는 신뢰구간**이 명시되었는가?
- 사례 증거라면: **생존자 편향**(성공 사례만 선택)은 없는가?
- "연구에 따르면"이라고 썼다면: **어떤 연구인지 특정 가능**한가?

### 출력 형식

```text
[주장 1] "핵심 주장 요약"
  → 근거 등급: A/B/C/D/E
  → 구체적 근거: 있음(출처) / 모호함 / 없음

[주장 2] ...
```

---

## Phase 4 — 논리 구조 검증 (Logic & Fallacy Check)

> 근거가 있더라도 **논리적 연결이 잘못**되면 결론은 틀릴 수 있다.

아래 **흔한 논리 오류** 해당 여부를 점검한다:

- [ ] **인과 오류** — 상관관계를 인과관계로 오인 ("A 후에 B가 일어났으므로 A가 원인")
- [ ] **생존자 편향** — 실패 사례를 무시하고 성공 사례만 근거로 제시
- [ ] **확증 편향** — 자기 주장을 뒷받침하는 증거만 선택적으로 인용
- [ ] **권위 호소** — "유명한 사람이 말했으므로 맞다"
- [ ] **성급한 일반화** — 소수 사례를 전체에 적용
- [ ] **허수아비 논증** — 상대 주장을 왜곡한 뒤 반박
- [ ] **거짓 이분법** — 두 가지 선택지만 제시하며 중간 대안을 무시
- [ ] **미끄러운 경사면** — 한 단계의 변화가 극단적 결과로 이어진다고 주장

### 출력 형식

```text
[논리 오류] 발견됨 / 발견되지 않음
[유형]      해당 오류 이름
[위치]      텍스트 내 해당 부분 요약
```

---

## Phase 5 — 시간 검증 (Temporal Validity Check)

> 정보에는 **유통기한**이 있다.

- 이 정보는 **언제 작성/발표**되었는가?
- 해당 분야에서 이후 **업데이트된 데이터나 반박 연구**가 있는가?
- 기술·시장·규제 등 **빠르게 변하는 분야**라면, 현재 시점에서도 유효한가?

### 출력 형식

```text
[작성 시점]  YYYY-MM 또는 추정 시기
[유효성]     현재 유효 / 부분 유효 / 만료 가능성 높음
[근거]       업데이트 필요 사유 1줄
```

---

## Phase 6 — 반대 시나리오 (Counter-Scenario Analysis)

이 글의 핵심 결론이 **틀릴 수 있는 조건**을 최소 2개 제시한다.

```text
만약 _____ 라면, 이 주장은 틀린다.
만약 _____ 라면, 정반대 결론이 나온다.
```

### 추가 질문

- 이 주장의 **전제 조건** 중 현실에서 깨지기 쉬운 것은?
- **다른 분야·다른 시장·다른 규모**에서도 동일하게 적용되는가?
- 저자가 **의도적으로 생략**했을 가능성이 있는 반례는?

---

## Phase 7 — 적용성 판단 (Applicability Assessment)

건재의 현재 상황·프로젝트·맥락에 비추어 검토한다.

- 이 인사이트가 **내 업종, 팀 규모, 리소스 수준**에 맞는가?
- 적용하려면 **어떤 전제 조건**이 충족되어야 하는가?
- 적용했을 때 **예상되는 리스크**는 무엇인가?
- **부분 적용**이 가능한가? (전체 도입 vs 핵심만 차용)

### 출력 형식

```text
[적용 가능성] 높음 / 조건부 / 낮음
[필요 전제]    충족해야 할 조건
[추천 액션]    구체적 다음 단계 1~2개
```

---

## 최종 판정 — 신뢰도 스코어카드

모든 Phase를 종합하여 아래 스코어카드를 출력한다.

```text
┌─────────────────────────────────────────────┐
│           비판적 검증 스코어카드              │
├──────────────────┬──────────────────────────┤
│ 출처 신뢰도      │ ●●●●○ (4/5)             │
│ 이해충돌         │ ●○○○○ (1/5) ⚠️          │
│ 근거 품질        │ ●●●○○ (3/5)             │
│ 논리 건전성      │ ●●●●● (5/5)             │
│ 시간 유효성      │ ●●●○○ (3/5)             │
│ 반대 시나리오    │ 2개 제시됨               │
│ 적용 가능성      │ 조건부                   │
├──────────────────┼──────────────────────────┤
│ 종합 판정        │ ⚠️ 조건부 수용            │
│                  │ 이해충돌 확인 필요        │
└──────────────────┴──────────────────────────┘
```

### 종합 판정 기준

| 판정 | 조건 |
| ---- | ---- |
| ✅ **수용** | 모든 항목 3/5 이상, 논리 오류 없음 |
| ⚠️ **조건부 수용** | 1~2개 항목 2/5 이하이거나 경미한 논리 오류 |
| ❌ **보류/기각** | 3개 이상 항목 2/5 이하이거나 심각한 논리 오류 |

---

## 저장 규칙

- **✅ 수용** → 분야별 폴더에 인사이트 저장 + 원문 링크 첨부
- **⚠️ 조건부 수용** → `_review/` 폴더에 저장, 추가 검증 필요 태그 부착
- **❌ 보류/기각** → 저장하지 않음. 기각 사유만 1줄 기록

### 저장 형식

```text
## [YYYY-MM-DD] 인사이트 제목
- 출처: 원문 링크
- 판정: ✅/⚠️/❌
- 핵심 인사이트: 1~2문장 요약
- 적용 액션: 구체적 행동 1~2개
- 주의사항: 검증에서 발견된 약점
```
