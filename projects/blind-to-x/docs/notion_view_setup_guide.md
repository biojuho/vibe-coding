# 📋 Notion 검토 뷰 설정 가이드

> Notion API는 뷰(View) 생성을 지원하지 않으므로, 아래 절차를 따라 Notion UI에서 직접 설정합니다.
> 이 가이드는 채널 발행보다 `검토 판단`과 `피드백 기록`을 먼저 쉽게 만드는 데 초점을 둡니다.

## 1. 📋 검토 워크플로우 (Board View)

> 목적: 상태 이동과 검토량 파악

### 설정 절차
1. DB 우상단 `+` → **Board** 선택
2. 이름: `📋 검토 워크플로우`
3. **Group by**: `상태`
4. 그룹 순서: `검토필요` → `승인됨` → `보류` → `반려` → `발행완료`
5. **Card preview / 표시 속성**
   - 콘텐츠
   - 토픽 클러스터
   - 최종 랭크 점수
   - 위험 신호
   - 반려 사유
6. **정렬**: `최종 랭크 점수` Descending

### 사용법
- 카드 열람 전 보드에서 `위험 신호`와 `반려 사유` 분포를 먼저 봅니다.
- `검토필요`에서 열어 보고, 판단이 끝나면 상태만 이동합니다.

---

## 2. 🧠 검토 큐 (Table View)

> 목적: 무엇을 봐야 하는지 한 화면에서 확인

### 표시 권장 컬럼
- `콘텐츠`
- `원본 소스`
- `토픽 클러스터`
- `감정 축`
- `최종 랭크 점수`
- `운영자 해석`
- `근거 앵커`
- `검토 포인트`
- `피드백 요청`
- `위험 신호`
- `트윗 본문`
- `뉴스레터 초안`
- `반려 사유`

### 필터 / 정렬
- 필터: `상태` is `검토필요`
- 정렬 1: `최종 랭크 점수` Descending
- 정렬 2: `생성일` Descending

### 사용법
- 검토자는 `검토 포인트`와 `피드백 요청`만 먼저 읽고 판단합니다.
- 문제가 보이면 자유서술보다 먼저 `반려 사유`를 체크합니다.
- 코멘트가 필요할 때만 `메모`나 `검토 메모`를 보강합니다.

---

## 3. 🧾 반려 회고 (Table View)

> 목적: 어떤 실패 패턴이 반복되는지 축적

### 설정 절차
1. DB 우상단 `+` → **Table** 선택
2. 이름: `🧾 반려 회고`
3. **필터**
   - `상태` is `반려`
4. **표시 속성**
   - 콘텐츠
   - 토픽 클러스터
   - 위험 신호
   - 반려 사유
   - 운영자 해석
   - 검토 포인트
   - 생성일

### 사용법
- 반려 사유가 반복되는 조합이 보이면 프롬프트/게이트 개선 후보입니다.
- `위험 신호`와 `반려 사유`가 자주 어긋나면 경고 규칙이 과한 것입니다.

---

## 4. ✍️ 채널별 발행 후보 (Optional)

> 목적: 승인된 초안을 채널별로 나눠서 복사

### 권장 구성
- 뷰 이름: `✍️ 숏폼 후보`
  - 필터: `상태` is `승인됨`
  - 필터: `발행 플랫폼` contains `숏폼`
  - Card preview: `트윗 본문`

- 뷰 이름: `🧵 Threads 후보`
  - 필터: `상태` is `승인됨`
  - 필터: `발행 플랫폼` contains `Threads`
  - Card preview: `Threads 본문`

- 뷰 이름: `📨 뉴스레터 후보`
  - 필터: `상태` is `승인됨`
  - 필터: `발행 플랫폼` contains `뉴스레터`
  - Card preview: `뉴스레터 초안`

> 자동 발행은 기본 범위가 아닙니다. 승인된 항목만 사람이 골라 수동으로 발행합니다.

---

## 5. ✅ 설정 전 확인

Notion 스키마 점검:

```bash
py -3 scripts/notion_doctor.py --config config.yaml
```

검토용 컬럼/옵션 동기화:

```bash
py -3 scripts/sync_notion_review_schema.py --config config.yaml --apply
```

기존 카드 리뷰 컬럼 백필:

```bash
py -3 scripts/backfill_notion_review_columns.py --config config.yaml --apply
```

## 6. 검토자가 실제로 보는 순서

1. `운영자 해석`
2. `근거 앵커`
3. `검토 포인트`
4. `피드백 요청`
5. `위험 신호`
6. 초안 본문

이 순서대로 보면 "어디를 봐야 하지?"가 아니라 "이 한 군데가 문제다" 수준으로 빠르게 판단할 수 있습니다.
