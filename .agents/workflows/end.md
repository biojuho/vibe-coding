# /end — 세션 종료 워크플로우

// turbo-all

## 실행 절차

1. 이번 세션에서 에이전트가 수행한 작업을 정리한다:
   - 변경된 파일 목록과 각 변경 이유
   - 주요 의사결정 및 근거
   - **(추가)** 변경된 코드에 대한 QA/QC 워크플로우 통과 여부 명시

2. 현재 미완성/버그 이슈를 정리한다:
   - 알려진 버그나 문제점
   - 각 이슈의 심각도 (🔴 높음, 🟡 보통, 🟢 낮음)

3. 다음 세션에서 이어할 TODO 항목을 작성한다:
   - 우선순위별 정렬
   - 예상 작업량 표기 (S/M/L)

4. 위 1~3번 내용을 Knowledge Base에 저장한다:
   - 관련 KI가 있으면 업데이트
   - 없으면 새로운 KI 아티팩트로 생성

5. 사용자에게 최종 요약 보고서를 제출한다.
