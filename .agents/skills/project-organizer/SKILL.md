# 🗂️ Project Organizer Skill

프로젝트 폴더 구조를 `.agents/rules/project-rules.md`의 규칙과 비교하여 정리하고, 변경 내용을 Knowledge Base에 기록합니다.

## Instructions

### Phase 1: 스캔 및 분석

1. `.agents/rules/project-rules.md`를 읽어 현재 폴더 구조 규칙을 파악한다.
1. 워크스페이스 루트의 전체 파일 트리를 스캔한다 (`list_dir` 사용).
1. 다음 항목을 점검한다:
   - 규칙에 정의되지 않은 루트 레벨 폴더/파일
   - 잘못된 위치에 있는 파일 (예: 루트에 있는 프로젝트 관련 파일)
   - 빈 폴더 또는 중복 파일
   - `_archive/`로 이동해야 할 비활성 프로젝트
   - `.tmp/` 정리 가능한 임시 파일

### Phase 2: 이동 계획 제안

1. 규칙에 맞지 않는 파일/폴더의 목록과 제안된 이동 경로를 표로 정리한다:

```markdown
| 현재 위치 | 제안 위치 | 이유 |
|-----------|-----------|------|
| /old-project/ | /_archive/old-project/ | 비활성 프로젝트 |
```

1. **반드시 사용자에게 승인을 요청한다** — 파일 이동은 승인 없이 절대 실행하지 않는다.

### Phase 3: 실행

1. 승인된 항목에 대해 파일/폴더를 이동한다.
1. 이동된 파일의 import 경로, 참조 경로를 검색하여 수정한다:
   - `grep_search`로 이전 경로 참조를 찾는다.
   - 해당 파일들의 경로를 새 위치로 업데이트한다.
1. 이동 후 영향받는 프로젝트의 빌드/테스트가 정상인지 확인한다.

### Phase 4: 기록

1. 변경 내용을 Knowledge Base에 기록한다:
   - 이동된 파일 목록
   - 수정된 참조 경로
   - 정리 전후 구조 비교

## Edge Cases

- **서버 실행 중 이동 금지**: MCP 서버나 dev server가 실행 중인 폴더는 이동 불가. 먼저 종료를 안내한다.
- **Git 추적 파일**: `git status`를 확인하여 uncommitted changes가 있으면 먼저 커밋을 권고한다.
- **대규모 이동**: 10개 이상의 파일 이동 시 배치(batch) 단위로 나누어 진행한다.

## Trigger Phrases

다음과 같은 요청 시 이 스킬을 사용합니다:

- "폴더 정리해줘"
- "프로젝트 구조 점검해줘"
- "파일 정리 좀 해줘"
- "워크스페이스 청소해줘"
