# 내부용 도구 프레임워크

## 필수 확인 항목

### 1. 사용자 프로필
- **규모**: 1명(본인) / 소규모 팀(2~5명) / 팀 전체(10+명)
- **기술 수준**: 개발자 / 기술 친화적 비개발자 / 비기술자
- **사용 빈도**: 매일 / 주 1~2회 / 월 1회 / 이벤트 기반

### 2. 현재 수동 프로세스
- 지금 이 작업을 어떻게 하고 있는가
- 수동 프로세스의 Pain point (시간, 실수, 반복성)
- 자동화하면 안 되는 부분이 있는가 (승인, 검토 등)

### 3. 성공 지표
- **시간 절약**: 기존 대비 몇 % 단축
- **오류 감소**: 수동 실수 제거
- **일관성**: 표준화된 출력물
- **확장성**: 데이터/사용자 증가 시 대응

### 4. 유지보수
- 누가 유지보수하는가 (본인 / 팀원)
- 문서화 수준 요구사항
- 모니터링/알림 필요 여부

## 설계 원칙

- **단순함 우선**: 내부 도구는 기능보다 안정성이 중요
- **문서 최소화**: 코드가 곧 문서 (self-explanatory naming)
- **안전장치**: 되돌릴 수 없는 작업에 확인 단계
- **로깅**: 무슨 일이 있었는지 추적 가능해야 함
- **점진적 확장**: 지금 필요한 것만 먼저, 나중에 확장
