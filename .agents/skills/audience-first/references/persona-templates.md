# 업종별 유저 페르소나 템플릿

## 템플릿 사용법

아래 템플릿을 기반으로 프로젝트에 맞게 수정하여 사용한다.
모든 필드를 채울 필요는 없다 — 프로젝트에 중요한 항목만 선택.

---

## 공통 페르소나 구조

```markdown
**이름**: [가상 이름]
**나이**: [연령대]
**직업**: [직군/직급]
**기술 수준**: [비기술자 / 기술 친화적 / 개발자]
**핵심 목표**: [이 제품/콘텐츠로 얻고 싶은 것]
**주요 불만**: [현재 해결 안 되는 것]
**사용 맥락**: [언제, 어디서, 어떤 디바이스로]
**의사결정 기준**: [무엇이 선택을 좌우하는가 — 가격, 편의성, 신뢰도 등]
```

---

## 업종별 예시

### 1인 창업자 / 프리랜서
- **기술 수준**: 중간 (노코드~로코드 경험)
- **핵심 목표**: 반복 업무 자동화, 시간 확보
- **주요 불만**: 좋은 도구는 비싸고, 무료 도구는 기능 부족
- **의사결정 기준**: 무료/저가, 빠른 셋업, 문서 불필요

### 스타트업 개발자
- **기술 수준**: 높음
- **핵심 목표**: 빠른 프로토타이핑, 생산성 향상
- **주요 불만**: 보일러플레이트 코드, 설정 지옥
- **의사결정 기준**: DX(개발자 경험), 커뮤니티, 확장성

### 비기술 팀원 (마케터/PM/디자이너)
- **기술 수준**: 낮음~중간
- **핵심 목표**: 데이터 기반 의사결정, 자동 리포팅
- **주요 불만**: 개발팀에 매번 요청해야 함
- **의사결정 기준**: UI 직관성, 학습 곡선, 즉시 가치 확인

### 학생/입문자
- **기술 수준**: 낮음
- **핵심 목표**: 학습, 포트폴리오 구축
- **주요 불만**: 튜토리얼과 실제 프로젝트 간 갭
- **의사결정 기준**: 무료, 쉬운 시작, 성취감

### 중소기업 관리자
- **기술 수준**: 낮음
- **핵심 목표**: 업무 효율화, 비용 절감
- **주요 불만**: IT 인력 부족, 기존 시스템 노후화
- **의사결정 기준**: ROI, 안정성, 고객 지원

### 콘텐츠 크리에이터
- **기술 수준**: 중간 (도구 활용 능숙)
- **핵심 목표**: 콘텐츠 생산 속도 향상, 품질 일관성
- **주요 불만**: 반복적인 편집/포맷팅 작업
- **의사결정 기준**: 시간 절약, 출력 품질, 워크플로우 통합
