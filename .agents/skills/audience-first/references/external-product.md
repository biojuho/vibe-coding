# 외부용 제품/서비스 프레임워크

## 필수 확인 항목

### 1. 타깃 유저 페르소나
- **인구통계**: 연령대, 직군/직급, 기술 수준
- **행동 패턴**: 어떤 디바이스/환경에서 사용하는가
- **의사결정자 vs 사용자**: 같은 사람인가, 다른 사람인가

### 2. 핵심 문제/페인포인트
- 현재 이 문제를 어떻게 해결하고 있는가 (수동, 경쟁 제품, 미해결)
- 이 문제가 얼마나 자주, 얼마나 심각하게 발생하는가
- 이 문제를 해결하면 유저에게 어떤 가치가 생기는가

### 3. 경쟁 대안 분석
- 직접 경쟁: 같은 문제를 푸는 제품
- 간접 경쟁: 다른 방식으로 같은 니즈를 충족하는 것 (ex. 스프레드시트)
- 우리의 차별점: 왜 경쟁 대안 대신 이것을 써야 하는가

### 4. 핵심 경험 (Core UX)
- 유저가 제품을 쓰면서 가장 "아하!" 하는 순간은 언제인가
- 첫 사용 시 5분 안에 가치를 느낄 수 있는가
- 재방문/재사용 동기가 있는가

### 5. 비즈니스 모델
- 무료 / 프리미엄 / 유료 / 내부 도구화
- 수익화 시점과 방법
- 유저 획득 채널

## 설계 체크리스트

- [ ] 온보딩 플로우가 있는가 (첫 사용자 가이드)
- [ ] 빈 상태(Empty State)가 설계되어 있는가
- [ ] 핵심 기능까지 3클릭 이내로 도달 가능한가
- [ ] 에러 메시지가 유저 친화적인가
- [ ] 모바일 대응이 필요한가
- [ ] 접근성(a11y)을 고려했는가
- [ ] 성능(로딩 속도)이 유저 기대에 부합하는가
