# 콘텐츠(글/영상/발표) 프레임워크

## 필수 확인 항목

### 1. 독자/시청자 페르소나
- **누구인가**: 직군, 경험 수준, 관심사
- **왜 이 콘텐츠를 보는가**: 문제 해결 / 호기심 / 업무 필요 / 엔터테인먼트
- **현재 지식 수준**: 초보 / 중급 / 전문가

### 2. 채널과 맥락
- **플랫폼**: 블로그, YouTube, 뉴스레터, SNS, 사내 위키, 발표
- **소비 환경**: 출퇴근 중 모바일 / 사무실 데스크톱 / 회의실
- **시간 예산**: 1분 스낵 / 5분 글 / 15분 영상 / 30분 발표
- **경쟁 콘텐츠**: 같은 주제로 이미 좋은 콘텐츠가 있는가

### 3. 목표 행동 (CTA)
- 콘텐츠를 본 후 독자가 해야 할 행동:
  - 제품 사용해보기
  - 의견/댓글 남기기
  - 공유하기
  - 더 깊은 콘텐츠로 이동
  - 구매/구독

### 4. 감정적 반응
- 독자가 콘텐츠 후 느꼈으면 하는 감정:
  - "나도 할 수 있겠다" (자신감)
  - "몰랐던 걸 알게 됐다" (인사이트)
  - "공유해야겠다" (공감/놀라움)
  - "이 사람/팀이 믿음직하다" (신뢰)

## 콘텐츠 유형별 구조

### 블로그/글
```
훅 (첫 2문장) → 문제 정의 → 해결/인사이트 → 실행 가능한 요약 → CTA
```

### 영상 스크립트
```
훅 (0~15초) → 맥락 설정 → 본론 (3~5 포인트) → 요약 → CTA + 구독 유도
```

### 뉴스레터
```
[HOOK] 한 줄 요약 → [STORY] 맥락 + 디테일 → [INSIGHT] 시사점 + CTA
```

### 발표/슬라이드
```
문제 (왜 여기 모였나) → 현황 → 제안 → 증거/데이터 → 다음 단계 → Q&A
```

## 톤 가이드

| 독자 수준 | 톤 | 전문 용어 | 예시 비율 |
|-----------|-----|----------|----------|
| 초보자 | 친절, 격려 | 최소 (쓰면 반드시 설명) | 높음 (매 개념마다) |
| 중급자 | 실용적, 효율적 | 적절히 사용 | 중간 (핵심만) |
| 전문가 | 동료 대화 | 자유롭게 | 낮음 (새로운 것만) |
