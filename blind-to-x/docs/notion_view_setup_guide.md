# 📋 Notion 뷰 설정 가이드

> Notion API는 뷰(View) 생성을 지원하지 않으므로, 아래 절차를 따라 Notion UI에서 직접 설정합니다.

## 📊 현재 DB 정보

- **데이터베이스**: 📋 X 콘텐츠 파이프라인
- **DB ID**: `2d8d6f4c-9757-4aa8-9fcb-2ae548ed6f9a`

---

## 1. 📋 발행 워크플로우 (Board View)

> **목적**: 콘텐츠 상태를 한눈에 파악하고 드래그로 상태 변경

### 설정 절차:
1. DB 우상단 `+` (Add a view) → **Board** 선택
2. 이름: `📋 발행 워크플로우`
3. **Group by**: `승인 상태`
4. 그룹 표시 순서: `검토필요` → `승인됨` → `발행완료` → `반려`
5. **표시 속성** (Card preview):
   - 콘텐츠 (제목)
   - 토픽 클러스터
   - 감정 축
   - 최종 랭크 점수
   - 원본 소스
6. **필터**: `상태` is not `Draft` (초안 단계 제외)
7. **정렬**: `최종 랭크 점수` Descending

### 사용법:
- 카드를 다른 컬럼으로 드래그하여 상태 변경
- `검토필요` → 내용 확인 후 `승인됨`으로 이동
- `승인됨` → 발행 후 `발행완료`로 이동

---

## 2. 🎴 X 트윗 큐 (Gallery View)

> **목적**: 승인된 트윗을 미리보기하고 빠르게 복사

### 설정 절차:
1. DB 우상단 `+` → **Gallery** 선택
2. 이름: `🎴 X 트윗 큐`
3. **Card preview**: `트윗 본문` (rich text 본문이 카드에 표시됨)
4. **표시 속성**:
   - 콘텐츠 (제목)
   - 트윗 본문
   - 토픽 클러스터
   - Source URL
5. **필터**: 
   - `승인 상태` is `승인됨`
   - `발행 플랫폼` contains `X` 또는 `twitter`
6. **정렬**: `최종 랭크 점수` Descending

### 사용법:
- 카드 클릭 → `트윗 본문` 복사 → X에 붙여넣기 발행
- 발행 후 `승인 상태`를 `발행완료`로 변경
- `트윗 링크`에 발행된 URL 기록

## 4. 🗓️ 콘텐츠 캘린더 (Calendar View)
