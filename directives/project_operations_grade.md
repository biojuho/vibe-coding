## 등급별 운영 지침

### Active 프로젝트

- CI 테스트 필수 (커밋 당)
- 커버리지 목표: >= 75%
- 의존성 보안 스캔: 주간
- 로그 모니터링: loguru + Telegram 알림
- MCP 서버: 상시 가동

### Maintenance 프로젝트

- CI 테스트: 월 1회 수동 실행
- 커버리지 유지만 (신규 테스트 불필요)
- 의존성: `npm audit` / `pip audit` 월 1회
- 알림: 에러 시에만
- MCP 서버: on-demand

### Frozen 프로젝트

- 테스트: 분기 1회 smoke test
- 의존성 업데이트 안 함 (보안 취약점 발견 시에만)
- CI 비활성화
- 문서에 "Frozen" 상태 명시
