# 코드 개선 지침 (Code Improvement Directive)

> 소스 코드를 스캔하여 품질 이슈를 식별하고 개선 사항을 제안합니다.

## 1. 목표

소스 코드 파일을 분석하여 가독성, 성능, 보안 이슈를 찾아내고, 각 이슈에 대해 수정된 코드 스니펫과 함께 실행 가능한 개선 사항을 제안합니다.

## 2. 입력

| 항목 | 필수 | 설명 |
|------|------|------|
| 파일/디렉토리 경로 | 필수 | 분석 대상 경로 |
| 심각도 필터 | 선택 | low / medium / high (기본값: 전체) |
| 카테고리 필터 | 선택 | readability / performance / security (기본값: 전체) |
| 출력 형식 | 선택 | json / markdown (기본값: json) |

## 3. 도구

- `execution/code_improver.py` — 결정론적 정적 분석 스크립트

## 4. 프로세스

1. **입력 검증**: 경로 존재 여부, 바이너리 파일 아닌지 확인
2. **스크립트 실행**: `python execution/code_improver.py <경로> --format json [--severity X] [--category Y]`
3. **JSON 결과 파싱**: 스크립트의 구조화된 출력을 파싱
4. **각 이슈에 대해**:
   - 관련 소스 코드 컨텍스트 읽기
   - 이 이슈가 **왜 중요한지** 설명 생성
   - **개선된 코드 스니펫** 생성 (LLM의 핵심 역할)
   - 프로젝트 맥락을 고려한 최종 심각도 판정
5. **결과 포맷팅**: 아래 출력 형식에 따라 사용자에게 제시

## 5. 출력 형식

각 이슈에 대해 다음 형식으로 출력:

```
### [심각도] 규칙ID: 이슈 설명
**카테고리:** readability | performance | security
**파일:** path/to/file.py:라인번호

**현재 코드:**
\`\`\`language
현재 문제가 있는 코드
\`\`\`

**개선된 코드:**
\`\`\`language
LLM이 제안하는 개선 코드
\`\`\`

**설명:** 왜 이 변경이 필요한지에 대한 상세 설명
```

## 6. 지원 언어

| 언어 | 확장자 | 분석 방법 |
|------|--------|-----------|
| Python | .py | AST 파싱 + 정규식 |
| JavaScript | .js, .jsx, .mjs | 정규식 기반 |
| TypeScript | .ts, .tsx | 정규식 기반 |
| Java | .java | 정규식 기반 |
| Go | .go | 정규식 기반 |
| C/C++ | .c, .h, .cpp, .cc, .hpp | 정규식 기반 |

## 7. 분석 카테고리

- **가독성 (Readability)**: 네이밍 규칙, 함수 길이, 복잡도, 중첩 깊이, 타입 힌트, 사용하지 않는 코드
- **성능 (Performance)**: 뮤터블 기본값, 루프 내 문자열 연결, 비효율적 패턴
- **보안 (Security)**: eval/exec, SQL 인젝션, XSS, 하드코딩된 시크릿, 버퍼 오버플로우

## 8. 예외 상황 (Edge Cases)

| 상황 | 처리 방법 |
|------|-----------|
| 바이너리 파일 (.exe, .dll 등) | 건너뛰기 + 메시지 |
| 미니파이된 파일 | 건너뛰기 + 경고 |
| 자동 생성된 코드 | 건너뛰기 + 메시지 |
| 매우 큰 파일 (>1MB) | 건너뛰기 + 경고 |
| 인코딩 오류 | UTF-8 → Latin-1 순서로 시도 |
| 빈 파일 | 건너뛰기 |
| 권한 오류 | 건너뛰기 + 경고 |

## 9. 종료 코드

- `0`: 분석 성공 (이슈 유무 무관)
- `1`: 오류 (파일 미존재, 치명적 오류)
- `2`: 분석 가능한 파일 없음
